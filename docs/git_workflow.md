# Git Workflow

Projektet er versionsstyret med Git og hostet på GitHub.

Repository:
https://github.com/abdulali29/Pratikprojekt

## Workflow

Udviklingsprocessen følger et simpelt Git workflow:

1. Ændringer implementeres i projektets kode
2. Filer tilføjes til staging area
3. Commit oprettes med beskrivende besked
4. Koden pushes til GitHub repository

Eksempel:

git add .
git commit -m "Add ETL pipeline"
git push origin main

## Formål

Git anvendes til:

- versionsstyring af kildekode
- dokumentation af udviklingshistorik
- sikker backup af projektet
- samarbejdsværktøj
